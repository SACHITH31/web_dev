# mv Command

## What is `mv`?

Used to move or rename files and folders.

---

# Syntax

```bash
mv source destination
```

---

# Rename Example

```bash
mv app.js main.js
```

Renames `app.js` to `main.js`.

---

# Move Example

```bash
mv app.js src/
```

Moves `app.js` into the `src` folder.

---

# Relative Path Example

```bash
mv app.js ../
```

Moves `app.js` one folder back.

---

# Multiple Files

```bash
mv app.js style.css src/
```

Moves multiple files into `src`.

---

# Rename Folder

```bash
mv backend server
```

Renames `backend` folder to `server`.

---

# Move Folder

```bash
mv backend project/
```

Moves `backend` into `project`.

---

# Multiple Commands Example

```bash
mkdir project && touch app.js && mv app.js project/ && ls project
```

### What Happens?

1. Creates `project`
2. Creates `app.js`
3. Moves `app.js` into `project`
4. Shows files inside `project`

---

# Important Point

Unlike `cp`, `mv` does not create duplicates.

After moving:

```bash
mv app.js src/
```

`app.js` no longer exists in the old location.

---

# Diagram

```text
Before:
project
├── app.js
└── src

After:
project
└── src
    └── app.js
```

---

# Difference Between cp and mv

| Command | Purpose |
| ------ | ------- |
| `cp` | Creates duplicate |
| `mv` | Moves original |

---

# Notes

- `mv` = Move
- Used for moving and renaming
- Original file changes location/name
- Useful for organizing files and folders
- Can also move folders using the same syntax
